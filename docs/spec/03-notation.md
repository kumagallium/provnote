# 記法と操作

Graphium のインライン操作は2つの記号に集約される。

---

## # = 意味を付ける（ラベル）

このブロックは「何」であるかを宣言する。`#` を打つとラベル候補のドロップダウンが出現し、1〜2文字 + Enter で確定（3打鍵）。確定するとテキストから `#xxx` が消え、バッジが表示される。

### コアラベル 5種

| 表示名 (en/ja) | 入力例 | 内部名 | PROV |
|---|---|---|---|
| Procedure / 手順 | `#手`, `#step` | [手順] | prov:Activity |
| Input / インプット | `#in`, `#材`, `#material` | [材料] | prov:Entity + used (material subtype) |
| Tool / ツール | `#ツ`, `#tool` | [ツール] | prov:Entity + used (tool subtype) |
| Output / アウトプット | `#out`, `#結`, `#result` | [結果] | prov:Entity + wasGeneratedBy |
| Attribute / 属性 | `#属`, `#attr` | [属性] | prov:Entity + 親への関係 |

コアラベルの表示名はドメインに依存しない汎用的な名称（Input / Output / Tool）を採用している。内部名（`[材料]`、`[結果]` 等）は実験科学をリードドメインとして開発した経緯から残っているが、ユーザーにはi18nにより汎用名で表示される。

5種すべてが PROV に直接マッピングされる。ラベルの役割は「このブロックは PROV 上で何であるか」を宣言すること。この一貫性を保つため、UI トリガーやテーブル操作の機能はラベルではなくブロック型（インデックステーブル）で実現する。

### エイリアス

`ALIAS_MAP` により、ユーザーは各ドメインの用語でラベルを付けられる。実験科学なら `#材料` `#試薬` `#装置`、ソフトウェア開発なら `#input` `#output` `#tool` など。エイリアスはすべてコアラベルに正規化される。

### フリーラベル

コアラベルにもエイリアスにも一致しないラベル（`#目的`、`#考察` 等）はフリーラベルとして登録される。PROV には変換されないが、バッジは表示され、知識ネットワーク上で見出しの意味を示す。

### 漸進的構造化

同じラベルでも入力形式によって PROV 出力の精度が自動的に変わる。

| 入力 | PROV 出力 |
|---|---|
| 段落 `Cu粉末 1g` | Entity { rdfs:label: "Cu粉末 1g" } |
| テーブル `名前\|量\|純度` | Entity { rdfs:label: "Cu粉末", Graphium:量: "1g" } |
| 画像ブロック | Entity { prov:type: "Image" } |

メモ書きレベルでも動作する。テーブルで書いた分だけ構造化の恩恵を受ける。

### 入力の3つの入口

1. `#` テキスト（主要な入口）
2. スラッシュメニュー `/手順`（ブロック + ラベル同時生成）
3. ドラッグハンドル（後付け）

どれも `LabelStore.setLabel(blockId, label)` を呼ぶだけ。

---

## @ = 別の何かを指す（参照）

別のブロックやノートへのリンクを作る。リンクタイプは `"reference"` の1種類のみ（Zettelkasten の「リンクはリンク」原則）。「支持する」「矛盾する」はリンク周辺のテキストが伝える。

### 「アドレス可能」と「候補に出る」は別の概念

すべてのブロックは blockId を持つのでアドレス可能。しかしオートコンプリートの候補に出るのは構造を持つものだけ。

| 対象 | 候補に出るか | 理由 |
|---|---|---|
| ノートタイトル / H2・H3 見出し | 常時 | スコープの「入口」 |
| ラベル付きブロック | ドリルダウン時 + 検索 | ラベルで名前がある |
| メディア（画像・PDF） | ドリルダウン時 + 検索 | ファイル名・ラベルでマッチ |
| ラベルなしの普通の段落 | 出ない | テキスト断片では選びようがない |

ラベルなしブロックは入口3（リンクコピー）でのみアドレス取得可。

### @ 参照の3つの入口

**入口1: ドリルダウン（場所を覚えている）**

`@Cu焼結/` と打つとそのノートの見出し一覧が出る。選ぶとラベル付きブロック一覧へ。各段階で候補は数件〜十数件。

**入口2: @ 内キーワード検索（場所を忘れた）**

`@SEM 800` のようにキーワードを入力すると、全ノート横断でラベル・ファイル名・テキスト・メディア名を検索し、結果を同じドロップダウン内に表示。ドリルダウンと検索は同じ入力欄で共存（`/` を含めばドリルダウン、含まなければ検索）。

**入口3: ブラウズ → リンクコピー → ペースト（じっくり探す）**

アセットブラウザや別ノートを見に行き、「リンクをコピー」→ 元のノートにペースト → 自動的に `@` メンションに変換。

### @ パスの内部表現

パス表記は人間用の表示名。実体は `blockId` + `noteId` で参照を保持する。見出しをリネームしてもリンクは壊れない。

## インデックステーブル（ブロック型）

「行がノートを表すテーブル」のカスタムブロック型。スラッシュメニュー `/インデックステーブル` で呼び出す。通常のテーブルとの唯一の違いは、行頭にノート作成アイコン（ドキュメント + 矢印）があること。

### 行に対する2つの操作

**操作1: アイコンクリック → 新規ノート作成**

行頭のアイコンをクリックすると、1列目の値をタイトルにしたノートが作成され、セルが `@` リンクに変換される。

**操作2: セルに @ 入力 → 既存ノートをリンク**

1列目のセルに `@` を入力すると、通常のオートコンプリートが動作し、既存ノートを検索してリンクできる。

### 行の3状態

| 状態 | 行頭表示 | 1列目の内容 |
|---|---|---|
| 未リンク | ノート作成アイコン | テキスト（試料名など） |
| 新規作成済み | ✓ | @ リンク（新規ノートへ） |
| 既存リンク済み | ✓ | @ リンク（既存ノートへ） |

### 用途

インデックステーブルは実験の試料管理に限らず、汎用的に使える:

- **試料管理:** 各行が試料ノート。条件列で比較。
- **文献管理:** 各行が論文の読書メモノート。著者・年・キーワード列。
- **装置管理:** 各行が装置マニュアルノート。型番・場所列。
- **実験シリーズ:** 各行が実験の記録ノート。目的・状態列。
