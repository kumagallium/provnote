# 画面構成と可視化

「書く面は静か、読む面が賢い」の「読む面」を担う。

---

## 画面レイアウト

```
┌────────────────────────────────────────────────────────┐
│ ヘッダー: ノートタイトル / 保存状態 / 設定              │

│ 最近の    │                        │ Graph PROV Chat 派生 │
```

## 左パネル: ナビゲーション

ノートはストレージプロバイダー（Google Drive / Local / Filesystem）にフラットに保存される（フォルダ階層は作らない）。Zettelkasten はネットワーク構造であり、ツリー型の親子関係に押し込めるべきではない。構造的なナビゲーションは `@` 参照・インデックステーブル・Graph タブが担う。

### 最近のノート

左パネル上部に最近使用したノート5件を表示。日常の作業動線はこれでカバーされる。

### ノート一覧

「ノート一覧を開く」をクリックすると、全ノートをテーブル形式で表示するデータベースビューが開く。

**表示列:**
- ノートタイトル（クリックで開く）
- 被参照数（何個のノートから `@` されているか）
- 含まれるラベル（`#手順` `#結果` 等のバッジ）
- 更新日

**ソート:**
- 被参照数で降順ソートすると、多くのノートから `@` されている MOC 的ノートが上に来る。試料ノート（被参照数1）やメモ（被参照数0）は下に。
- 更新日・作成日・タイトルでもソート可能。

**フィルタ:**
- ラベルフィルタ: `#結果` を含むノート、`#手順` を含むノート等
- テキスト検索: ノートタイトルで検索

被参照数による自然なランキングが、フォルダ構造なしで「どれが概要でどれが末端か」を伝える。ユーザーが意識しなくても、使い方が構造を作る。

### アセットタブ（実装済み）

メディア（画像・PDF）+ ラベル付きコンテンツの横断一覧。メディアは `usedIn` でノートと紐付けられ、メディアからノートを辿ることも可能。

## 右パネルの4タブ

### Graph — 知識ネットワーク

現在のノートを中心に、`@` 参照とノート間派生で繋がるノート群を可視化。Cytoscape.js + fcose レイアウト。2ホップ BFS で近隣ノートを取得。

**ノードの種類:**

| ノード | ソース | 表示 |
|---|---|---|
| 現在のノート | アクティブなファイル | 中心に配置、ハイライト |
| @ 参照先ノート | knowledgeLinks | 1ホップ隣接 |
| 派生元・派生先 | derivedFromNoteId / provLinks | 矢印付き |
| 2ホップ先 | 隣接ノートの knowledgeLinks | 薄く表示 |

**エッジの種類:**

| エッジ | 視覚的表現 |
|---|---|
| @ 参照 | 破線、色なし |
| 派生 | 実線、矢印付き |

MOC 的なノート（`@` 参照が密集）はグラフ上でハブとして自然に浮かび上がる。

**操作:** ノードクリック → そのノートを開く。右クリック → 「新しいタブで開く」「Split View で開く」。

### PROV — 来歴グラフ

現在のノート内のラベル付きブロックから生成される PROV を DAG として可視化。Cytoscape.js + ELK レイアウト。

- **Activity（手順）:** `[手順]` ラベル付きスコープから生成。長方形。
- **Entity（Input・Tool・Output・Attribute）:** 対応するラベル付きブロックから生成。楕円。
- **エッジ:** `prov:used`, `prov:wasGeneratedBy`, `prov:wasInformedBy`

### Chat — AI 対話（実装済み）

見出しスコープに紐づく AI チャット。継続的な複数ターン対話。MCP ツール統合により外部ツール呼び出しも可能。

- 「ノートに反映」（ブロック挿入）、「別ノートとして派生」、「ブロック置換」の3操作
- AI による構造化提案はチャットの中でのみ行う。エディタ上では割り込まない。

### 派生元 — 読み取り専用

派生ノート作成時のみ出現。元ノートの全体を読み取り専用で表示。

## エディタ上のインジケーター

エディタ本体は「静か」を保ちつつ、控えめなインジケーターで状態を知らせる。

| インジケーター | 表示条件 | 動作 |
|---|---|---|
| ラベルバッジ | ラベルが付いたブロック | クリックでラベル変更・削除 |
| チャットマーク | ScopeChat が存在するスコープの見出し | クリックで右パネル Chat タブを開く |
| 派生バッジ | 派生元 or 派生先がある場合 | クリックで派生先ノートを開く |
| @ インラインリンク | テキスト内の @ 参照 | クリックで参照先にジャンプ |

---

## backlink の表示

- **表示場所:** 右パネルの Graph タブ下部に backlink 一覧セクション
- **内容:** 参照元のノート名、スコープ名、参照を含むテキストの抜粋
- **クリック:** 参照元ノートを開き、該当ブロックにスクロール
- **自動生成:** `@` 参照を作成すると、ターゲット側に backlink が自動追加

## 機能の出現条件

UI の複雑さはユーザーが書いた量に比例する。

| ユーザーの行為 | 出現する機能 |
|---|---|
| メモだけ書く | テキストエディタ + 右パネル(空) |
| ラベルを付ける (#手順) | + PROV ノード、ラベルバッジ |
| テーブルで構造化 | + 構造化 Entity (見た目は変わらない) |
| @ で別ノートを参照 | + ネットワークグラフ、backlink |
| AI チャットを開く | + Chat タブ、チャットマーク |
