# コアコンセプト

provnote の設計を貫く概念体系。

---

## 統一モデル: Zettelkasten + オプショナル PROV

- `@` 参照で全スコープが Zettel としてネットワーク化される（全ノート共通、ラベル不要）
- `#` ラベルで PROV ロールを付与する（構造化したいノートだけ、オプション）
- ラベルを付けなければ PROV には一切関与せず、普通のメモ帳として使える

---

## スコープ

provnote の全機能はスコープを単位に動作する。

**スコープ = 見出しブロック + 配下のブロック群。** ノート全体も1つのスコープ。

### 見出しレベルとスコープの関係

| レベル | 役割 | スコープとしての扱い |
|---|---|---|
| H1 | ノートタイトル。1ノートに1つ。 | ノート全体のスコープ。`@ノート名` で参照。 |
| H2 | 実験手順や大きなセクション。 | 主要なスコープ単位。`#手順` ラベルの主な対象。PROV Activity に対応。 |
| H3 | H2 の中のサブステップ。 | H2 の子スコープ。`@ノート名/焼結/温度制御` のように参照可能。 |
| H4 以下 | 細かい小見出し。 | スコープとしては扱わない（`@` 参照の候補に出ない）。文書の見た目の整理に使う。 |
| 通常ブロック（段落、テーブル等） | コンテンツ。 | スコープではないが、ラベルを付ければ `@` 参照の候補に出る。blockId でアドレス可能。 |

H2 と H3 が provnote の主要なスコープ階層。H4 以下はスコープとしては機能せず、純粋にドキュメントの見出し構造として使う。通常ブロックはスコープを持たないが、`#結果` 等のラベルを付ければ「名前のある存在」として `@` 参照の候補に出現する。

### スコープに対する3つの操作

- **入れ子** — H2 の中に H3。見出し階層。
- **分岐** — 試料ごとに別ノート / ノート派生。
- **接続** — PROV リンク / `@` 参照。スコープ間の関係。

PROV Activity はスコープ（H2 または H3 の `[手順]` ラベル付き見出し + 配下）に対応する。AI チャットはスコープに紐づく。ノート派生はスコープから新ノートを作る。

---

## スコープ = Zettel

あらゆるスコープは `@` で参照可能な Zettel（情報カード）として機能する。

```
@ノート名                       → ノート全体
@ノート名/焼結                  → H2 スコープ
@ノート名/焼結/密度             → 特定のラベル付きブロック
```

ラベルの有無に関係なく、どのスコープも参照可能。PROV 化しない雑多なメモのスコープも、ラベル付きノートの結果ブロックも、同じように `@` で指して別のノートから利用できる。

---

## MOC (Map of Contents)

MOC は特別なノート種別ではない。`@` 参照が密集したノートが自然に MOC として機能する。

典型的な用途:
- 研究テーマ全体の俯瞰（「@実験1 の結果は @実験2 と整合し、@文献A の報告を支持する」）
- 複数実験の結果を集約した比較テーブルやチャート
- 試料管理の実験概要ノート（→ [06-samples.md](06-samples.md)）

---

## ノートの種類はない — ラベル密度のグラデーション

| ラベル密度 | 典型的な内容 | 動作 |
|---|---|---|
| ラベルなし | 雑多なメモ、アイデア、日記 | Zettel のみ（@ 参照可能、PROV なし） |
| 一部ラベル | 手順だけ構造化したメモ | Zettel + 部分 PROV |
| フルラベル | 来歴を追跡したいノート | Zettel + 完全 PROV |
| @ 参照主体 | MOC、研究テーマまとめ | Zettel 集約ビュー |

### 実験の3層構造

実験科学では、ノートの役割が自然に3層に分かれる。

| 層 | 典型的な内容 | MaiML 対応 |
|---|---|---|
| 手順テンプレート | 再利用可能な標準手順 | Protocol |
| 実験概要 (MOC) | 1回の実験の計画・比較・考察 | Instruction |
| 試料ノート | 各試料の実際の記録（全コアラベルを使用） | Result |

3つとも同じ `ProvNoteDocument`。`@` 参照の向きとラベルの密度で役割が決まる。手順テンプレートは複数の実験から `@` で参照されることで Class として再利用される。詳細は [06-samples.md](06-samples.md)。

## メディアレジストリ

画像・PDF もノート横断の Zettel として管理する（Capacities のメディアデータベースに着想）。

### 動作原理

- **アップロード時:** ユーザーがエディタに画像をドロップすると、裏側でメディアレジストリに登録される（体験は変わらない）。ファイルは Google Drive に独立保存され、ノート側の画像ブロックには `mediaId`（参照ID）だけを持つ。
- **参照:** 同じメディアを別のノートから `@` で参照できる。実体は1つ、表示は複数。
- **PROV:** メディアに `#結果` や `#属性` ラベルを付ければ、PROV Entity として来歴が追える。

### 実験科学での活用

| 種類 | 具体例 | 横断的に見る価値 |
|---|---|---|
| 画像 | SEM画像、光学顕微鏡写真 | 全実験の SEM 画像をギャラリーで一覧表示 → 共通パターンの発見 |
| グラフ | XRD パターン、応力-ひずみ曲線 | 異なる実験の XRD を重ね合わせ比較 |
| PDF | 参考文献、装置マニュアル | 1つの論文を複数ノートから参照（再アップロード不要） |
| データファイル | CSV、Excel | 生データへの永続的リンク |

### アセットブラウザ

左パネルのファイル一覧と同階層に「アセット」タブとして配置する。メディア（画像・PDF）だけでなく、コアラベルが付いたコンテンツもノート横断で一覧表示する。

**メディア表示:**
- グリッド形式のサムネイル一覧
- フィルタ: ファイル種別（画像 / PDF / データ）、ノート別、日付順
- 操作: クリックでプレビュー。「リンクをコピー」で内部リンクをクリップボードに。

**ラベル付きコンテンツ表示:**
- ラベル別のリスト表示。例: 「#結果」で絞ると全ノートの結果ブロックが一覧に並ぶ
- 「#手順」で絞ると、過去にやった手順の一覧が見える
- 各項目にはソースノートへのリンクと、テキスト内容の抜粋が表示される

**活用場面:**
- 全実験の SEM 画像をギャラリーで一覧表示 → 共通パターンの発見
- 「#結果」一覧で全実験の密度結果を横断的に俯瞰
- 「#手順」一覧で過去の手順を検索し、新しい実験に再利用
- 特定の論文 PDF を探して別ノートに引用
- 来歴表示: メディアやラベル付きブロックを選択すると、PROV 上のソースが表示される
