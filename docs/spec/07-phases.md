# 段階的開発計画

provnote の設計を段階的に実装するロードマップ。技術的な依存関係ではなく「誰がどのくらいの頻度で使うか」で順序を決めている。この順序は設計思想「Zettelkasten が基盤、PROV はオプション」とも一致する。

---

## Phase 一覧

| Phase | 対応機能 | 誰が使うか | ユーザーに見える変化 |
|---|---|---|---|
| **1** | 「双方向リンク」 | 全員・毎日 | @ と打つとリンクが生まれ、ネットワークグラフが見える |
| **2** | 「PROV-DM」 | 実験記録時 | #手順 等のラベルで右パネルに来歴グラフが自動表示 |
| **3** | （新規） | 複数試料実験時 | インデックステーブルから試料ノートを作成。@ プレビューで俯瞰 |
| **4** | 「AI統合」 | 全員・頻繁 | スコープに紐づくAIチャット。結果をノートに反映可能 |
| **5** | （新規） | 蓄積後 | アセットブラウザで画像・PDF・ラベル付きコンテンツを横断一覧 |

Phase 1-3 はすべてオプショナルフィールド追加のみ。マイグレーション不要。

---

## Phase 1: @ 参照 + 知識ネットワーク

**全員が毎日使う。Zettelkasten 基盤。**

ユーザーに届く価値: `@` を打つと他のノートや見出しが候補に出て、リンクが生まれる。ネットワークグラフで繋がりが見える。backlink 自動表示。

作るもの:
- `@` オートコンプリート（ノート一覧 + 見出し一覧）
- `knowledgeLinks: BlockLink[]` を ProvNotePage に追加
- backlink 自動生成
- 右パネル Graph タブ（Cytoscape.js + fcose）

設計判断: リンクタイプは `"reference"` の1種類のみ。@ パスは表示用、実体は blockId + noteId。

---

## Phase 2: # ラベル + PROV ビュー

**実験記録する人が使う。Phase 1 のネットワークに「因果」が加わる。**

ユーザーに届く価値: `#手順` `#結果` と付けるだけで、右パネルに来歴グラフが自動表示。フォーム入力なし。

作るもの:
- `#` オートコンプリート
- `labels` と `provLinks` を ProvNotePage に追加
- コアラベル4種（[手順] / [使用するもの] / [結果] / [属性]）+ エイリアス
- generator.ts（PROV-JSON-LD 計算。ビューとして毎回再生成）
- 右パネル PROV タブ（Cytoscape.js + ELK）

設計判断: PROV はリードオンリーのビュー。`provLinks`（DAG）と `knowledgeLinks`（循環OK）は分離。

> **Phase 1 + 2 で基盤完成。** 双方向リンク + PROV-DM が実現する。

---

## Phase 3: 試料管理 (Plan/Provenance 分離)

**Phase 1-2 の小さな延長。インデックステーブル + @ プレビュー。**

ユーザーに届く価値: `/インデックステーブル` で条件テーブルを作成。行頭のアイコンをクリックすると試料ノートが作成され、セルが `@` リンクに自動変換。セルに `@` を入力すれば既存ノートもリンクできる。実験概要ノート（MOC）に比較テーブル・チャート・考察を置き、試料ノートには個別の記録を書く。`@` リンクをクリック → プレビュー表示。

作るもの:
- インデックステーブル カスタムブロック型（行頭にノート作成アイコン）
- `@` 参照のプレビュー/ピーク表示（@ クリック → ポップオーバー）

作らないもの:
- sampleScope カスタムブロック型（不要）
- ノート内のタブ UI（不要）
- ネストされたエディタ（不要）

設計判断: TSJ2024 ポスターの構造に回帰。実験概要ノート = Plan Graph、試料ノート = Provenance Graph。詳細は [06-samples.md](06-samples.md)。

---

## Phase 4: AI チャット + 派生

**全員が頻繁に使う。@ と # があるとコンテキストが豊かになる。**

ユーザーに届く価値: スコープを選んでAIに質問。回答を「ノートに反映」or「別ノートとして派生」。対話履歴はスコープに保存。

作るもの:
- ScopeChat データモデル
- 右パネル Chat タブ
- 「ノートに反映」/ 「派生」UI
- Split View + パンくずナビ

設計判断: 構造化提案はチャット内のみ。エディタには割り込まない。

---

## Phase 5: アセットブラウザ

**蓄積が増えてから価値が出る。**

ユーザーに届く価値: 画像・PDFがノート横断で一覧表示・検索可能。`#結果` で絞れば全実験の結果一覧、`#手順` で絞れば手順一覧。同じ論文PDFを複数ノートから参照。

作るもの:
- メディアレジストリ（固有 ID、Drive 独立保存）
- 画像ブロックの `mediaId` 参照化
- アセットブラウザ（左パネル。メディア + ラベル付きコンテンツの横断一覧）
- メディアの `@` 参照対応

設計判断: メディアも Zettel として扱う。書く体験は変わらない（D&Dで画像挿入、裏側で自動登録）。
